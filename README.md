# Legacy Projects

This repository contains a curated archive of older projects, technical studies, course exercises, experiments, and proof-of-concept applications developed throughout my software development journey.

It exists to preserve my learning history while keeping my GitHub profile better organized and focused on current, relevant, and actively maintained repositories.

## Overview

Throughout my development path, I have built many projects to practice programming concepts, explore frameworks, complete courses, solve technical challenges, and experiment with different technologies.

Some of these repositories were valuable during my learning process but no longer represent my current standards, architecture decisions, or development practices. Instead of deleting them, I chose to keep them here as a historical archive.

This repository reflects part of my technical evolution as a developer.

## Why this repository exists

The main goal is organization.

By grouping older and smaller repositories here, my GitHub profile becomes easier to navigate and highlights the projects that better represent my current skills, professional focus, and software engineering standards.

This repository also works as a timeline of my progress, showing how my knowledge, code style, and technical decisions have evolved over time.

## Maintenance Status

Some of them may contain outdated dependencies, older framework versions, deprecated patterns, or incomplete documentation. They are kept mainly for reference and historical purposes.
